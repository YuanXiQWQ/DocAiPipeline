# DocAI Pipeline — 项目流程与任务规划

> 基于客户邮件（2026-03-03 ~ 2026-03-04）整理的完整业务流程与系统实现计划。

---

## 一、业务背景

**客户公司**：Terra Drvo d.o.o.（塞尔维亚 Ljukovo），从事欧洲橡木（欧橡）原木采购、加工（刨切、锯切）、表板生产与销售。

**核心痛点**：大量纸质单据需要人工录入 Excel，耗时且易出错。希望通过扫描件自动识别 → 自动填入 Excel 模板。

---

## 二、客户业务流程（原木全生命周期）

### 阶段 1：进口采购

**触发**：从克罗地亚（Hrvatske Šume 林业局）或其他供应商采购原木。

**报关公司**发来一组文件（每批一组）：
1. **CR_*.pdf** — 进口税款缴纳凭证
2. **Deklaracija_*.pdf** — 报关单（JCI 海关申报单）
3. **Racun.pdf** — 商业发票（供应商开具）
4. **SKEN.*.pdf** — 扫描件合集（多页 PDF，包含商业发票、EUR.1 产地证、检疫证书、CMR 运输单、运费发票等）

**目标输出**：将这些文件中的关键信息抽取后填入 → **`纸质发票记录表`** Excel

| 纸质发票记录表字段 | 来源文件 |
|---|---|
| 所属人、公司名 | 固定值（根据业务配置） |
| 供应商名称 | 商业发票(Racun) / SKEN 中的发票 |
| 性质（进口/国内） | 报关单(Deklaracija) |
| 名称（原木/运费等） | 商业发票 / 运费发票 |
| 日期 | 发票日期 |
| 发票号 | 发票编号 |
| 编号 | 内部编号（可能需要规则生成） |
| FSC（是/非） | EUR.1 证书或发票备注 |
| 供应商/备注 | 供应商名称 |
| 外币金额 | 发票总金额 |
| 数量（m³） | 发票中的体积数量 |

### 阶段 2：原木入库 — 检尺

**触发**：原木到货后，库管人员手工检尺。

**涉及文件**：
- **克罗地亚Ad公司原木检尺及入库单-库管填报2.pdf** — 手写检尺单（Ad公司供应的原木）
- **克罗地亚Hrvatske原木对方码单及检尺入库单.pdf** — 包含两部分：
  - 第1部分：供应商（Hrvatske Šume）出库码单（对方提供）
  - 第2部分：我方库管的检尺入库单

**检尺数据**：编号、长度（米）、径级/直径（厘米）→ 计算体积（m³）

**客户期望操作**：
1. 将供应商出库码单扫描件 → 转为 Excel → 提取编号、长、直径、m³ → 填入 `原木出入库` 表的对应位置
2. 将我方检尺入库单扫描件 → 转为 Excel → 用编号做 VLOOKUP 与供应商数据核对
3. **由库管将供应商的数量与我方检尺数量填到表3（验收入库单）中**

**目标输出**：→ **`原木出入库-3-3.xlsx`** 的 `数据源表` sheet

| 数据源表关键字段 | 来源 |
|---|---|
| 日期 | 入库日期 |
| 车间 | 大锯车间（固定） |
| 工序 | 来料 |
| 木种 | 欧橡 |
| 货物名称 | 原木 |
| 序号/批次号 | 检尺单批次号 |
| 送货客户 | 供应商名称 |
| 车牌号 | CMR 运输单 |
| 包号/根号 | 每根原木编号 |
| 等级 | F1/F2/厚皮等 |
| 长度 | 检尺数据 |
| 直径 | 检尺数据 |
| 数量 m³ | 根据长×直径计算 |

### 阶段 3：原木出库/领用

**触发**：大锯车间从仓库领用原木进行加工。

**涉及文件**：
- **原木领用出库表.pdf** — 手写的领用出库表

**客户期望**：扫描件自动转为 Excel → 填入 `原木出入库` 表（工序=出库）

### 阶段 4：刨切加工

**触发**：原木在大锯车间领用后，切成大方，送入蒸煮池浸泡 3-5 天，然后上刨切机。

**涉及文件**：
- **刨切木方入池表.pdf** — 手写入池记录
- **刨切木方上机表.pdf** — 手写上机记录

**客户期望**：扫描件 → Excel → 填入 **`刨切木方入池与上机表3-3.xlsx`**

**业务说明**：
- 入池表和上机表是一线员工每天手写统计，交给库管录入 Excel
- 刨切生产 1.2MM 和 2.0MM 厚度的表板
- 表板数量到下一工序（剪切）打包后才统计

### 阶段 5：表板打包与销售

**触发**：刨切产出的表板经剪切打包后成为成品。

**涉及文件**：
- **表板打包报表.pdf** — 手写的打包报表（一线人员每天报给库管）

**客户期望**：扫描件 → Excel → 填入 **`表板统计 - 20260304.xlsx`**

### 阶段 6：整体报表汇总

**最终汇总表**：**`新公司LKV表板统计.xlsx`**

- 所有前面各环节的 Excel 数据最终汇入这个整体报表
- 通过**数据透视表**查看库存的进销存情况
- 这是管理层查看的核心报表

## 四、系统模块划分

### 模块 A：进口单据识别与录入
- **输入**：报关公司发来的 PDF 组（CR, Deklaracija, Racun, SKEN）
- **处理**：VLM 抽取 → 字段映射 → 填入纸质发票记录表
- **难度**：★★★☆☆（印刷体为主，结构化较好）
- **当前状态**：✅ MVP 已实现，准确率 90%+

### 模块 B：原木检尺单识别与录入
- **输入**：手写检尺单扫描件（编号、长度、径级）
- **处理**：VLM/OCR 抽取表格数据 → 计算体积 → VLOOKUP 核对 → 填入原木出入库表
- **难度**：★★★★☆（手写表格，数据量大，需精确数字识别）
- **当前状态**：⚠️ 未专项实现

### 模块 C：原木出库/领用单识别
- **输入**：手写领用出库表扫描件
- **处理**：VLM 抽取 → 填入原木出入库表（工序=出库）
- **难度**：★★★☆☆
- **当前状态**：⚠️ 未专项实现

### 模块 D：刨切入池/上机表识别
- **输入**：手写入池表、上机表扫描件
- **处理**：VLM 抽取 → 填入刨切木方入池与上机表
- **难度**：★★★☆☆
- **当前状态**：⚠️ 未专项实现

### 模块 E：表板打包报表识别
- **输入**：手写打包报表扫描件
- **处理**：VLM 抽取 → 填入表板统计表
- **难度**：★★★☆☆
- **当前状态**：⚠️ 未专项实现

### 模块 F：Excel 自动填充引擎
- **功能**：读取 Excel 模板 → 根据抽取数据自动填入对应单元格 → 保留原有公式和格式
- **依赖**：openpyxl / xlsxwriter
- **难度**：★★★★☆（需要理解客户 Excel 模板的公式和数据关系）
- **当前状态**：❌ 未实现

### 模块 G：Web 前端（上传 + 复核 + 导出）
- **功能**：上传 PDF → 查看识别结果 → 人工复核修正 → 导出到 Excel
- **当前状态**：❌ 未实现

### 模块 H：整体报表汇总
- **功能**：各模块数据汇入 LKV 整体报表 → 数据透视
- **当前状态**：❌ 未实现（可能不需要自动化，客户已有公式）

## 六、关键技术决策

| 决策项 | 选择 | 原因 |
|--------|------|------|
| VLM 模型 | gpt-4.1-mini | 成本低、多语种好 |
| Excel 处理 | openpyxl | 保留公式和格式 |
| 手写 OCR | VLM 表格模式 | 无需单独训练 OCR 模型 |
| 文档检测 | 全页模式（MVP）→ YOLO（后续） | 避免 contour 误切 |
| 前端 | React + TailwindCSS | 现代 UI |
| 后端 | FastAPI (Python) | 与 AI 模块同语言 |
